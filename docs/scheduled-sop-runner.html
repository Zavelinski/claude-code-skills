scheduled-sop-runner for Claude Code

scheduled-sop-runner is a Claude Code skill that turns a repeatable task or SOP into a quiet, self-checking scheduled routine. It runs on a cadence, checks whether the run actually succeeded, and only pings you when something fails or a success condition is not met. No news is good news.

What it does

You define the SOP, a success predicate, and a cadence. Each run logs to an append-only run log, evaluates its own success predicate, and stays silent on success. You get notified only on failure, with the run log attached.

Install

Option 1, Claude Code plugin marketplace (recommended):

/plugin marketplace add Zavelinski/claude-code-skills
/plugin install scheduled-sop-runner@claude-code-skills

Option 2, manual fallback: clone the source repo and run its installer. The marketplace path registers any hooks through the Claude Code consent UI; the manual installer edits ~/.claude/settings.json directly and is blocked by auto-mode. Prefer Option 1.

How do I use it?

  • Define the SOP and its success predicate once.
  • Set a cadence (cron or interval).
  • Get pinged only when a run fails its own check.

Frequently asked questions

How is scheduled-sop-runner different from a cron job?
A cron job runs on a schedule and tells you it ran. scheduled-sop-runner also checks whether the run actually succeeded against a success predicate you define, and notifies you only on failure. No news means the run passed.
What is a success predicate?
A verifiable condition each run must satisfy, for example "the deploy returned HTTP 200" or "the report file is newer than yesterday". The runner evaluates it after every run and pings you only if it is not met.
Where does it keep the run log?
An append-only run log records each run, its result, and the success predicate outcome. You can audit the full history without wading through successful runs.
