content-repurpose-engine for Claude Code

content-repurpose-engine is a Claude Code skill that turns one source asset into many platform-native posts. Give it a blog post, transcript, video, or long note and it extracts the core idea once, then re-packages it per platform: LinkedIn, X thread, Instagram, short-form video script, newsletter, YouTube, Reddit, source-faithful with no invented facts.

What it does

It extracts the core idea once, then adapts format, length, and tone to each platform. It stays source-faithful: no invented facts, no hype added. The default set is 8 to 10 posts.

Install

Option 1, Claude Code plugin marketplace (recommended):

/plugin marketplace add Zavelinski/claude-code-skills
/plugin install content-repurpose-engine@claude-code-skills

Option 2, manual fallback: clone the source repo and run its installer. The marketplace path registers any hooks through the Claude Code consent UI; the manual installer edits ~/.claude/settings.json directly and is blocked by auto-mode. Prefer Option 1.

How do I use it?

  • Give it one source asset (URL, file, or pasted text).
  • It extracts the core idea and outputs 8-10 platform-native posts.
  • Each post is ready to paste into its platform.

Frequently asked questions

Does content-repurpose-engine invent facts?
No. It is source-faithful. It extracts the core idea from your asset and re-packages it per platform without adding facts, claims, or hype that were not in the source.
Which platforms does it output?
The default set is 8-10 posts: LinkedIn, X thread, Instagram, short-form video script, newsletter, YouTube, Reddit, and more. Each is formatted for its platform native length and tone.
What counts as a source asset?
A blog post, transcript, video, or long note. Give it a URL, a file, or pasted text. It extracts one core idea and repackages it across platforms.
