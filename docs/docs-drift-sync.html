docs-drift-sync for Claude Code

docs-drift-sync is a Claude Code skill that keeps docs in sync with the code that changed. When you change code, docs silently go stale: a renamed parameter, a removed flag, a changed signature, an example whose output no longer matches. This maps the changed surface in a diff to where the docs mention it, detects drift, and proposes the minimal edits.

What it does

It is scoped to drift, not doc generation. It does not write new docs from scratch. It finds where the docs reference what you just changed, shows what is now wrong, and proposes the smallest edit that fixes it.

Install

Option 1, Claude Code plugin marketplace (recommended):

/plugin marketplace add Zavelinski/claude-code-skills
/plugin install docs-drift-sync@claude-code-skills

Option 2, manual fallback: clone the source repo and run its installer. The marketplace path registers any hooks through the Claude Code consent UI; the manual installer edits ~/.claude/settings.json directly and is blocked by auto-mode. Prefer Option 1.

How do I use it?

  • After a code change with a diff, run /docs-drift-sync.
  • Review the drift map and the proposed minimal edits.
  • Apply or reject each edit.

Frequently asked questions

Does docs-drift-sync generate new documentation?
No. It is scoped to drift. It maps what you changed to where the docs mention it, finds what is now wrong, and proposes minimal edits. It does not write docs from scratch.
What counts as drift?
Any doc mention that no longer matches the code: a renamed symbol, a removed flag, a changed signature, an example whose output no longer matches, a referenced file that moved.
How does it find the doc mentions?
It takes the changed surface from your diff, symbols, flags, and endpoints, and searches the docs for references to each, then checks whether the reference still matches the current code.
