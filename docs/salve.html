salve for Claude Code

salve is a Claude Code skill that ships and persists your work in one word. Say salve and it runs the whole close-out: persist context, commit, push, open and merge a PR to your main branch, trigger and confirm the deploy, fast-forward your local checkout, and report evidence.

What it does

One word runs the full ship routine in order: persist context across three layers, push and PR-and-merge to main, confirm the deploy, fast-forward the local checkout, and report commit, deploy, and rollback evidence. Generic public version with EN default and a pt-br variant.

Install

Option 1, Claude Code plugin marketplace (recommended):

/plugin marketplace add Zavelinski/claude-code-skills
/plugin install salve@claude-code-skills

Option 2, manual fallback: clone the source repo and run its installer. The marketplace path registers any hooks through the Claude Code consent UI; the manual installer edits ~/.claude/settings.json directly and is blocked by auto-mode. Prefer Option 1.

How do I use it?

  • When your work is ready: salve.
  • It persists, commits, pushes, PRs and merges, deploys, and reports.
  • Variant salve doc skips redeploy for doc-only changes.

Frequently asked questions

What does salve do in one word?
It runs the full close-out: persist context, commit, push, open and merge a PR to your main branch, trigger and confirm the deploy, fast-forward your local checkout, and report commit, deploy, and rollback evidence.
Does salve deploy by default?
Yes. salve deploys by default. Use salve doc to skip redeploy for doc-only changes, or salve sem deploy to push and persist without merging to production.
Is this the same salve I can install from the marketplace?
The marketplace ships a generic public salve with an EN default and a pt-br variant. It is the public version of the personal ship routine, not anyone private config.
