adversarial-verify for Claude Code

adversarial-verify is a Claude Code skill that proves a code change works by trying to break it. Given a change, it re-derives what the change is supposed to do, writes targeted tests that try to falsify that claim, runs them, and returns a binary PASS / FAIL / INCONCLUSIVE verdict backed by real executed output, never prose alone.

What it does

It differs from a prose code review. Instead of saying "looks correct", it produces a test that would have failed before the change and passes after, plus edge cases that attack the new behavior, and shows you the run output. No run, no verdict: it never claims PASS from reading alone.

Install

Option 1, Claude Code plugin marketplace (recommended):

/plugin marketplace add Zavelinski/claude-code-skills
/plugin install adversarial-verify@claude-code-skills

Option 2, manual fallback: clone the source repo and run its installer. The marketplace path registers any hooks through the Claude Code consent UI; the manual installer edits ~/.claude/settings.json directly and is blocked by auto-mode. Prefer Option 1.

How do I use it?

  • After a fix or feature: run /adversarial-verify.
  • It writes falsification tests in your existing test framework and runs them.
  • You get a binary verdict with the executed output and an explicit not-covered list.

Frequently asked questions

How is adversarial-verify different from a code review?
A code review emits prose opinions. adversarial-verify is falsification-first and execution-backed: it writes tests that try to break the change, runs them, and returns PASS/FAIL/INCONCLUSIVE with the output that proves the verdict. No run means no verdict.
What does PASS require?
PASS needs both: every falsification test passes, and at least one test actually exercises the change (it would have failed on the pre-change code). A green suite that never touches the change is not a pass.
Does it modify my source code to make tests pass?
No. Tests adapt to the code, never the reverse. If the code is wrong, the verdict is FAIL.
