skill-security-scan for Claude Code

skill-security-scan is a Claude Code skill that vets a third-party skill before it runs as you. A skill is untrusted instructions executing with your privileges, and a skill that ships a hook is code that runs every turn, yet almost nobody reads one before installing. This statically scans a SKILL.md, its hooks, and the settings.json it would register, and returns an ALLOW / REVIEW / BLOCK verdict with the exact lines to remove.

What it does

It never executes the scanned skill. It reads the manifest, any hook scripts, and the settings merge the installer would perform, flags injection, network access, filesystem scope, and prompt-injection vectors, and tells you what to delete before you install.

Install

Option 1, Claude Code plugin marketplace (recommended):

/plugin marketplace add Zavelinski/claude-code-skills
/plugin install skill-security-scan@claude-code-skills

Option 2, manual fallback: clone the source repo and run its installer. The marketplace path registers any hooks through the Claude Code consent UI; the manual installer edits ~/.claude/settings.json directly and is blocked by auto-mode. Prefer Option 1.

How do I use it?

  • Point it at a skill repo or SKILL.md: /skill-security-scan Zavelinski/claude-code-godmode.
  • Read the verdict and the flagged lines.
  • Install only if ALLOW, or after you remove the flagged lines for REVIEW/BLOCK.

Frequently asked questions

Does skill-security-scan run the skill it scans?
No. It is static and defensive. It reads the SKILL.md, hook scripts, and settings.json merge, flags risks, and returns a verdict. It never executes the target skill.
What is the verdict contract?
ALLOW means no high-risk hooks or settings; REVIEW means flags worth removing before install; BLOCK means a clear bypass or exfiltration vector. Each verdict lists the exact lines to remove.
Why do I need this before installing a Claude Code skill?
A skill runs as you with your privileges, and a skill that ships a UserPromptSubmit hook injects text into every prompt. Most users install skills without reading them. This skill reads them for you, statically, before install.
